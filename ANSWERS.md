# Ответы на вопросы

## Пуш-уведомления

Технически реализуемо, но требует нескольких шагов:

1. **PWA** — добавить `manifest.json` и Service Worker (Next.js это поддерживает через `next-pwa`)
2. **Web Push API** — браузерный стандарт, работает в Chrome/Safari/Firefox
3. **VAPID-ключи** — генерируются один раз, хранятся в Vercel env vars
4. **Supabase Edge Function** — серверная функция, которая отправляет пуш при новом сообщении/лайке/комментарии
5. **Хранение подписок** — нужна таблица `push_subscriptions` в БД

Сложность: средняя. Время: ~1 день. Пока не реализовано.

## Звонки, видео-звонки, трансляции

**Да, всё реализуемо.** Варианты:

### Аудио/видео звонки

| Сервис | Плюсы | Минусы |
|---|---|---|
| **Daily.co** | Простая интеграция, готовый SDK | Платно после 10,000 мин/мес |
| **Agora** | Мощный SDK, низкий latency | Сложнее в интеграции |
| **Stream Video** | Щедрый free tier, хороший SDK | Относительно новый |
| **Twilio Video** | Надёжный, хорошая документация | Дорого |
| **WebRTC (DIY)** | Бесплатно | Нужны STUN/TURN серверы, сложно |

**Рекомендация:** Daily.co или Stream Video — быстрая интеграция, есть React SDK.

### Трансляции (один-ко-многим)

| Сервис | Описание |
|---|---|
| **Mux** | Профессиональный стриминг, RTMP + HLS, хорошая цена |
| **Cloudflare Stream** | Дешевле Mux, простая интеграция |
| **LiveKit** | Open-source WebRTC, можно self-host |

**Рекомендация:** Mux для стриминга. Реально сделать в 2-3 дня.

**Когда делать:** после запуска основного функционала, когда будет понятна аудитория.

## Доступ администратора к сообщениям

Текущий статус: RLS запрещает чтение сообщений третьим лицам.

**Как реализовать:**
- Отдельный роут `/api/admin/messages` на сервере, использует `SUPABASE_SERVICE_ROLE_KEY` (обходит RLS)
- Доступ только для role = admin/creator
- Все обращения логируются

Это правильный подход — admin видит сообщения только через server-side API, не через клиент. Реализуем по запросу.
